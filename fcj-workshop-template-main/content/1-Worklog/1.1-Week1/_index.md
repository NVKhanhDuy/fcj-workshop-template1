### Week 1 Learning Objectives:

* Study foundational knowledge of AWS Cloud, the 2025 AWS Free Tier policy, and tools supporting cost management during the use of AWS services.
* Gain a clear understanding of the mechanisms of identity management, access control, and permission granting for AWS services through AWS IAM and IAM Roles.
* Become familiar with the AWS Cloud9 development environment and core infrastructure services including Amazon VPC, Amazon EC2, Amazon S3, and Amazon RDS.
* Practice deploying basic AWS resources to build a foundation for practical exercises and projects in the following weeks.

### Week 1 Results Achieved:

#### 1. AWS Free Tier 2025 Learning
* Gained a clear understanding of AWS Free Tier plans such as Always Free, 12 Months Free, and Trials.
* Learned about the usage limits of common AWS services and how to avoid unnecessary costs.
* Became more aware of monitoring resource consumption to manage expenses effectively.

#### 2. Cost Management with AWS Budgets
* Understood how AWS Budget helps track and control spending.
* Set up budgets to monitor usage and maintain cost visibility.
* Configured email notifications to receive alerts when thresholds were reached.

#### 3. AWS Support Basics
* Learned about the different support options offered by AWS.
* Understood the process of creating and managing support cases.
* Differentiated support categories such as billing, technical assistance, and service limit requests.

#### 4. Identity and Access Management (IAM)
* Learned the purpose of IAM users, groups, and policies in AWS.
* Understood the principle of least privilege for secure access control.
* Gained practical knowledge on how to manage access to AWS resources.

#### 5. IAM User and Permission Management Practice
* Created IAM users and groups for access control.
* Assigned policies to users and groups based on required permissions.
* Verified whether the configured permissions worked as intended.

#### 6. IAM Role Understanding
* Distinguished between IAM users and IAM roles.
* Learned how trust policies and permission policies work together.
* Understood how AWS services can be granted access without directly using access keys.

#### 7. IAM Role Assignment Practice
* Created IAM roles and attached them to AWS services.
* Tested the access flow through the assigned roles.
* Confirmed that resources could be accessed correctly through role-based permissions.

#### 8. AWS Cloud9 Experience
* Set up a cloud-based development environment.
* Used the browser-based editor and integrated terminal efficiently.
* Became familiar with the basic development tools available in AWS Cloud9.

#### 9. Amazon S3 and Static Website Hosting
* Learned the object storage model used by Amazon S3.
* Created S3 buckets and managed uploaded objects.
* Configured static website hosting and verified the website endpoint.

#### 10. Amazon VPC Knowledge
* Gain a clear understanding of the important role of VPC in building AWS network infrastructure.
* Learned about core components such as CIDR blocks, subnets, route tables, and internet gateways.
* Recognized the difference between public and private subnets.

#### 11. VPC Deployment Practice
* Created a custom VPC for the lab environment.
* Set up public subnets and configured network routing.
* Verified network connectivity within the AWS environment.

#### 12. Amazon EC2 Understanding
* Learned the concept of virtual servers on AWS.
* Gained knowledge about AMIs, instance types, key pairs, and security groups.
* Understood the basic process of launching and managing EC2 instances.

#### 13. EC2 Deployment Practice
* Created an EC2 instance and configured its security settings.
* Connected to the server through SSH.
* Verified the instance was running correctly.

#### 14. Amazon RDS Understanding
* Learned about the managed database service model on AWS.
* Studied components such as database instances, database engines, and endpoints.
* Understood the importance of security and access control in databases.

#### 15. Amazon RDS Deployment Practice
* Created an RDS MySQL instance for the lab setup.
* Configured essential database parameters.
* Set up security groups to allow proper connectivity.
* Verified the connection between the EC2 instance and the RDS instance.
